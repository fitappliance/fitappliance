Methodology

Last updated: 18 April 2026

FitAppliance is a static decision-support tool for Australian appliance sizing. This page documents our calculation order, data transformations, and publication constraints.

1. Fit Calculation Order

  • Input cavity dimensions (width, height, depth) are validated client-side.
  • Per-brand clearance rules are applied before fit checks.
  • Doorway checks use delivery-width margin logic from generated doorway pages.
  • Products failing any mandatory dimension are excluded from fit results.

2. Data Sources

  • Core model data: Australian Government Energy Rating register snapshots in `public/data/appliances.json`.
  • Clearance rules: FitAppliance maintained `public/data/clearance.json` with explicit defaults.
  • Rebates: state-level references in `public/data/rebates.json`.
  • Brand organization metadata: `public/data/brands/metadata.json` (only when verified links exist).

3. Programmatic Content Rules

  • No generated page may include fake ratings, fake review counts, or fabricated citations.
  • Schema blocks are validated as strict JSON before publish.
  • Internal links are static HTML links to ensure crawler visibility.
  • All generated pages include update metadata via `article:modified_time`.

4. Operational Cadence

Automated pipelines regenerate structured pages, sitemap, RSS, image sitemap, and link graph reports. Editorial checks run in tests before deployment.
